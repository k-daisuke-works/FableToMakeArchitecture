# 公開前チェックリスト

content-reviewer はこの4観点で資料を検証する。観点を追加・修正したいときは
このファイルを直接編集する(スキル本文の変更は不要)。

## 1. 事実正確性

- [ ] モデル名・機能・料金・数値・日付・規制の記述が現時点の情報と一致している
      (WebSearch で 1 点ずつ照合する)
- [ ] すべての事実の主張に出典 URL と参照日が併記されている
- [ ] 「〜できる」という能力の主張に誇張がない(できる条件・限界も述べているか)

## 2. 医療・福祉分野での表現

- [ ] 医療行為の助言と読める表現がない(「診断に使える」「〜すれば治る」等)
- [ ] 冒頭に免責の定型文が入っている(masters/base-outline.md の文面)
- [ ] 個人情報・患者情報を生成AIに入力する場面を、注意喚起なしに推奨していない

## 3. 著作権・引用

- [ ] 引用には出典が明記され、引用の範囲が主従関係を保っている
- [ ] 出所不明・権利不明の画像参照がない
- [ ] 他者のスライド・図表の流用がない

## 4. 聴衆への適合(非エンジニアの医療・福祉従事者)

- [ ] 説明なしで使われている IT 専門用語がない(初出時に平易な言い換えがある)
- [ ] 例え・事例が医療・福祉の現場業務に即している
- [ ] 台本がスライド番号と対応し、時間配分の目安が書かれている

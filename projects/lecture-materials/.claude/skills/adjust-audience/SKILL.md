---
name: adjust-audience
description: 既存の講義資料の聴衆・難易度・トーン・時間を調整した派生版を作るとき。「今回は介護職向けに易しくして」「30分版にして」「管理職向けのトーンにして」という依頼で使う。
---

対象: $ARGUMENTS

1. 元にする講義(`lectures/<日付-テーマ>/`)と調整方針(聴衆像・時間・トーン)を特定する。
   $ARGUMENTS は `tools/request-form.html` が生成した要件定義書(【対象の資料】【聴衆】…
   形式)の場合もある。不明なら 1 回だけ確認する。ただし要件定義書の「未回答(〜してよい)」
   はユーザーが推測を許可した回答として扱い、確認せず進める。
2. 派生版は**元を上書きせず**、新しいディレクトリ `lectures/<新YYYYMMDD-テーマ>/` に作る
   (例: `20260810-genai-basics-kaigo/`)。
3. 時間が変わる場合は、まず `outline.md` の時間配分を再計算し、削る章・残す章を決める。
   その outline を確定させてから slides / script を追随させる(順序を守る)。
4. 用語・例えを聴衆に合わせて調整する:
   - カタカナ IT 用語 → 日常語に置換、または現場業務の例えを添える
   - 専門概念 → 対象職種の業務(介護記録、レセプト、ケアカンファレンス等)に例え直す
   - トーン指定(管理職向け等)があれば、リスク管理・導入判断の観点を厚くする
5. `/publish-check` を実行し、元資料との主な違い(削った章・変えた例え・時間配分)と
   チェック結果を報告する。

## 注意

- 出典・参照日・免責文は派生版でも維持する。易しくする過程で削らない。
- 元資料のディレクトリには一切書き込まない。

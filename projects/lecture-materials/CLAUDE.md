# lecture-materials — 講義資料作成

医療・福祉分野の非エンジニア従事者向け「生成AIの基礎知識」講義の資料一式
(講義構成・スライド・台本)を生成・改訂する体制。ソフトウェア開発は行わない。

## 成果物の形式

- スライド: **Marp 形式 Markdown**(`slides.md`)。標準出力は PDF、必要に応じて pptx にエクスポート。
- 構成案(`outline.md`)・台本(`script.md`): プレーン Markdown。
- 形式を変更する場合(例: pptx 直接作成に移行)は、この節だけを書き換える。
  各スキルは「CLAUDE.md の形式節に従う」と参照しているため、他は修正不要。

```bash
npx @marp-team/marp-cli slides.md -o slides.pdf    # PDF エクスポート
npx @marp-team/marp-cli slides.md -o slides.pptx   # pptx エクスポート
```

## ディレクトリ規約

- 講義一式は `lectures/<YYYYMMDD-テーマ>/` に置く(`outline.md`・`slides.md`・`script.md`)。
- マスター構成テンプレートは `masters/base-outline.md`。**直接編集は人間の承認を得てから**。
  各講義はこれを下敷きにするが、マスター自体は勝手に書き換えない。

## スキルの使い分け

依頼は自由文のほか、要件定義フォーム(`tools/request-form.html` をブラウザで開いて
生成した【テーマ】【聴衆】…形式のテキスト/保存した .md)でも受け付ける。

- 新しい講義の資料を作る → `/new-lecture`
- 既存資料を最新動向に更新する → `/update-lecture`
- 聴衆・時間・難易度を調整した派生版を作る → `/adjust-audience`
- 公開・配布の前 → **必ず `/publish-check`**

## 常に守ること

- 事実の主張(モデル名・数値・料金・規制・統計)には**出典 URL と参照日**を併記する。
  陳腐化しやすい情報は WebSearch で当日確認してから書く。
- 医療行為の助言と読める表現(「診断に使える」「〜すれば治る」等)を書かない。
  スライド冒頭に免責の定型文(masters/base-outline.md 参照)を入れる。
- 画像・引用は権利確認できるもののみ使う。出所不明の画像参照を書かない。
- 用語は非エンジニアの医療・福祉従事者に通じる言葉を使う。カタカナ IT 用語には
  現場業務に即した例えを添える。

## 完了の定義

`/publish-check` で content-reviewer の「要修正」がゼロになり、**人間が最終承認する**まで
公開可としない。公開可否の判断は常にユーザーが行う。

## 言語

やりとり・成果物はすべて日本語。ファイル名・識別子は英語ケバブケース。

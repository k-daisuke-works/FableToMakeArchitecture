---
name: verify
description: 変更後の動作確認をするとき。「動くか確認して」「テスト通る?」という依頼で使う。整形→リント→テスト→起動確認を定型手順で行う。/develop の検証ステップからも呼ばれる。
---

1. `ruff format .` を実行する。
2. `ruff check .` を実行し、指摘があれば修正して再実行する。
3. `test-runner` エージェントに `pytest -x -q` の実行を委譲し、失敗要約を受け取る。
   生ログをメイン会話に持ち込まない。
4. 失敗があれば修正して手順3を再実行する(失敗が解消するまで繰り返す。
   同じテストで3回続けて失敗したら停止して人間に報告する)。
5. 起動確認を行う(CLAUDE.md「コマンド」節の起動コマンド)。起動コマンドが
   TODO のまま未確定の場合はスキップし、その旨を報告に明記する。
6. 結果を「整形/リント/テスト(通過数)/起動」の各1行で報告する。

## 注意

- テストの失敗を「既存の問題」と判断して見送らない。今回の変更と無関係だと
  確認できた場合のみ、報告にその根拠を書いたうえで続行する。

# stock-investment — 株式運用リサーチ

元手資金と運用方針(対象市場・期間・目的・リスク許容度)から、具体的な銘柄・株数・
目安金額・タイミングまで落とし込んだ提案書を作成する個人向けリサーチ体制。
**実際の発注執行・証券会社API連携は行わない。情報の提示のみ。**

## 3つの必須ルール(プロトタイプ品質でも非交渉)

1. **免責文言**: すべてのレポート冒頭に次の定型文を入れる。
   > 本資料は情報提供のみを目的とした参考情報であり、投資助言ではありません。
   > 記載の株価・指標はリアルタイムではなく、取得時点のものです。
   > 投資の最終判断はご自身の責任で行ってください。
2. **出典と取得日時**: 株価・指標・業績などすべての数値に情報源URLと取得日時を併記する。
   数値が取得できなかった項目は「不明」と書く(推測で埋めない)。
3. **事実と推測の区別**: 取得した情報(事実)とエージェントの判断(推測)を
   見出しや表記で明確に分ける。

## ディレクトリ規約

- 提案書: `reports/proposals/<YYYY-MM-DD>-<方針名>.md`
- スクリーニング結果: `reports/screening/<YYYY-MM-DD>-<条件名>.md`
- 見直しレポート: `reports/reviews/<YYYY-MM-DD>-review.md`
- 通貨・取引時間は対象市場に応じて表記する(日本株=円、米国株=米ドル+現地時間、など)。

## スキルとエージェントの使い分け

依頼は自由文のほか、要件定義フォーム(`tools/request-form.html` をブラウザで開いて
生成した【元手資金】…形式のテキスト/保存した .md)でも受け付ける。

- 運用方針の確定からポートフォリオ提案書まで → `/portfolio-proposal`
- 条件で候補銘柄を絞り込む → `/stock-screening`
- 保有銘柄の定期見直し・乗り換え提案 → `/portfolio-review`
- 個別銘柄・市場・ニュースの Web 調査は必ず `stock-researcher` エージェントに委譲する
  (検索結果やページ本文をメイン会話に持ち込まない。トークン効率のため)。

## 人間確認の節目

①運用方針の確定時(/portfolio-proposal の冒頭)②最終提案書の提示時。
それ以外は自律で進める。

## やってはいけないこと

- 発注・売買の実行、証券会社・取引所APIの呼び出し(curl/wget 含む)
- メイン会話で直接 WebSearch/WebFetch して生の検索結果・ページ本文を持ち込む
  (調査は stock-researcher 経由で要約だけ受け取る)
- 出典のない数値・取得日時のない株価を書く
- 「必ず上がる」など断定的な利益保証表現
- .env・秘密情報の読み取り

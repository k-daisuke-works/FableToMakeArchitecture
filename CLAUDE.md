# FableToMakeArchitecture

このリポジトリは「エージェント体制を設計・構築するメタエージェント」である。
ユーザーが開発したいもの(例:「ECサイトを開発するエージェント設計をして」)を指示すると、
`projects/<プロジェクト名>/` 配下に、そのプロジェクト専用の Claude Code エージェント環境
(CLAUDE.md・スキル・サブエージェント・settings.json)と、そのエージェントへ依頼内容を
整理して渡すためのプロジェクト別要件定義フォーム `tools/request-form.html` を生成する。

指示は自由文のほか、要件定義フォーム(`tools/requirements-form.html` をブラウザで開いて
生成した【プロジェクト名】【目的】【体制の種類】… 形式のテキスト)でも受け付ける。

## 最重要ルール

- ユーザーが「〜を開発するエージェント(体制)を設計して/作って」という趣旨の指示をしたら、
  **必ず `/design-agents` スキルを起動する**。自己流で直接ファイルを書き始めない。
- 既存の `projects/<名前>/` に対する改善・監査の依頼は `/review-design` スキルを使う。
- 生成物はすべて `projects/<プロジェクト名>/` 配下に置く。このリポジトリ直下の
  `.claude/`・`references/`・`templates/` はメタエージェント自身の構成物なので、
  プロジェクト生成時に上書きしない。

## ディレクトリ構成

```
CLAUDE.md            ← このファイル(メタエージェントの動作定義)
.claude/
  skills/            ← メタエージェントのスキル(design-agents, review-design)
  agents/            ← メタエージェントのサブエージェント(分析・設計・構築・レビュー)
references/          ← 設計知識ベース(設計パターン、ファイル形式リファレンス)
templates/           ← 生成物の雛形(CLAUDE.md・スキル・エージェント・要件定義フォームHTML)
tools/               ← 人間が使う道具(要件定義フォームHTML)
projects/            ← 生成されたプロジェクト別エージェント環境(成果物置き場)
```

## 設計の原則

- 設計判断の根拠は `references/design-patterns.md` に従う。過剰な構成(不要なサブエージェント
  の乱立)は品質を下げる。プロジェクトの規模・リスク・反復性に見合った最小構成を選ぶ。
- 生成するファイルの形式は `references/claude-code-formats.md` に厳密に従う。
- 生成後は必ず design-reviewer サブエージェントによるレビューを通し、結果をユーザーに報告する。
- モデル配分(`references/design-patterns.md`「モデル配分とトークン効率」)は生成物だけでなく
  メタエージェント自身のサブエージェント(.claude/agents/)にも適用する。

## 言語

ユーザーとのやりとり、および生成する CLAUDE.md・スキル本文は日本語で書く。
frontmatter の name などの識別子は英語ケバブケースにする。
